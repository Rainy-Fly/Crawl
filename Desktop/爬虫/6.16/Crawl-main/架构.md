# 爬虫系统架构文档

## 一、整体系统概述

本系统是一个基于Python、HTML、CSS、JavaScript的爬虫系统，采用前后端分离架构，后端使用Flask框架，前端采用原生Web技术栈。系统提供用户账号管理、爬虫配置、定时任务、爬虫执行、结果可视化等完整功能。

## 三、功能模块

### 1. 用户认证模块

**页面：sign.html**
- **功能**：用户注册、登录
- **流程**：
  1. 用户输入账号密码进行注册
  2. 用户输入账号密码进行登录
  3. 登录成功后跳转到index.html

### 2. 主功能模块

**页面：index.html**

#### 2.1 配置文件上传区
- **功能**：上传爬虫配置文件和时间配置文件
- **上传方式**：
  - 页面文本框直接编写JSON
  - 点击上传按钮上传JSON文件

#### 2.2 已有配置文件查看区
- **功能**：展示用户已上传的配置文件
- **交互**：
  - 滚动列表展示所有配置文件
  - 点击配置文件可查看详细内容
  - 支持编辑修改配置文件

#### 2.3 示例配置文件区
- **功能**：提供示例配置文件参考
- **交互**：
  - 点击按钮显示示例配置文件内容
  - 支持下载示例配置文件到本地

#### 2.4 爬虫执行区
- **功能**：启动爬虫任务
- **流程**：
  1. 选择一个爬虫配置文件
  2. 可选时间配置文件（定时执行）
  3. 不选时间配置则立即执行
  4. 点击启动按钮进入爬虫池

#### 2.5 爬虫状态显示区
- **功能**：展示爬虫任务状态
- **状态类型**：未开始/进行中/已完成/已暂停
- **交互**：
  - 已完成任务提供两个操作按钮：
    1. 可视化：跳转到show.html查看结果
    2. 保存：将结果保存到数据库

### 3. 结果可视化模块

**页面：show.html**
- **左侧列表**：展示数据库中保存的爬虫结果和未保存的临时结果
- **中间区域**：可视化展示选中的爬虫结果
- **交互**：点击左侧列表项切换展示内容

### 2. 核心功能类

#### 2.1 User类
- **功能**：用户信息管理
- **属性**：
  - id: 用户唯一标识
  - username: 用户名
  - password: 密码（加密存储）
- **方法**：
  - register(): 用户注册
  - login(): 用户登录
  - get_user_info(): 获取用户信息

#### 2.2 ConfigFile类
- **功能**：配置文件管理
- **属性**：
  - id: 文件唯一标识
  - user_id: 所属用户ID
  - name: 文件名
  - type: 文件类型（爬虫配置/时间配置）
  - content: 文件内容（JSON格式）
  - create_time: 创建时间
- **方法**：
  - upload(): 上传配置文件
  - get_configs(): 获取用户所有配置文件
  - edit_config(): 编辑配置文件
  - delete_config(): 删除配置文件

#### 2.3 CrawlerTask类
- **功能**：爬虫任务管理
- **属性**：
  - id: 任务唯一标识
  - user_id: 所属用户ID
  - config_id: 关联的爬虫配置ID
  - time_config_id: 关联的时间配置ID（可选）
  - status: 任务状态（未开始/进行中/已完成/已暂停）
  - create_time: 创建时间
  - start_time: 开始时间
  - end_time: 结束时间
- **方法**：
  - start(): 启动爬虫任务
  - pause(): 暂停爬虫任务
  - resume(): 恢复爬虫任务
  - stop(): 停止爬虫任务
  - get_status(): 获取任务状态

#### 2.4 CrawlerResult类
- **功能**：爬虫结果管理
- **属性**：
  - id: 结果唯一标识
  - task_id: 关联的任务ID
  - content: 爬虫结果数据
  - create_time: 创建时间
  - is_saved: 是否保存到数据库
- **方法**：
  - save_to_db(): 保存结果到数据库
  - get_results(): 获取用户所有结果
  - visualize(): 可视化展示结果

## 五、数据库设计

### 1. 用户表（users）
| 字段名     | 类型         | 说明           |
|------------|--------------|----------------|
| id         | INTEGER      | 主键，自增     |
| username   | VARCHAR(50)  | 用户名，唯一   |
| password   | VARCHAR(255) | 加密后的密码   |
| create_time| DATETIME     | 创建时间       |

### 2. 配置文件表（config_files）
| 字段名     | 类型         | 说明           |
|------------|--------------|----------------|
| id         | INTEGER      | 主键，自增     |
| user_id    | INTEGER      | 外键，关联用户 |
| name       | VARCHAR(100) | 文件名         |
| type       | VARCHAR(20)  | 文件类型       |
| content    | TEXT         | 文件内容       |
| create_time| DATETIME     | 创建时间       |

### 3. 爬虫任务表（crawler_tasks）
| 字段名         | 类型         | 说明           |
|----------------|--------------|----------------|
| id             | INTEGER      | 主键，自增     |
| user_id        | INTEGER      | 外键，关联用户 |
| config_id      | INTEGER      | 外键，关联配置 |
| time_config_id | INTEGER      | 外键，关联时间配置 |
| status         | VARCHAR(20)  | 任务状态       |
| create_time    | DATETIME     | 创建时间       |
| start_time     | DATETIME     | 开始时间       |
| end_time       | DATETIME     | 结束时间       |

### 4. 爬虫结果表（crawler_results）
| 字段名     | 类型         | 说明           |
|------------|--------------|----------------|
| id         | INTEGER      | 主键，自增     |
| task_id    | INTEGER      | 外键，关联任务 |
| content    | TEXT         | 结果内容       |
| create_time| DATETIME     | 创建时间       |
| is_saved   | BOOLEAN      | 是否保存       |

## 七、系统流程

### 1. 用户注册登录流程
```
用户打开sign.html → 输入账号密码 → 点击注册/登录 → 后端验证 → 登录成功跳转到index.html
```

### 2. 爬虫执行流程
```
用户上传配置文件 → 选择配置文件 → 选择时间配置（可选） → 点击启动 → 后端创建任务 → 执行爬虫 → 展示状态 → 结果可视化/保存
```

### 3. 结果可视化流程
```
用户打开show.html → 选择结果 → 后端获取数据 → 前端可视化展示
```

## 八、扩展性考虑

1. **多爬虫引擎支持**：可扩展支持Scrapy、PySpider等爬虫框架
2. **分布式爬虫**：可扩展分布式爬虫集群
3. **更多可视化方式**：可扩展图表、地图等多种可视化方式
4. **数据导出**：可扩展导出为CSV、Excel、JSON等格式
5. **权限管理**：可扩展角色权限管理
6. **日志系统**：可完善系统日志和错误处理

## 九、性能优化

1. **异步任务处理**：使用Celery等异步任务队列处理爬虫任务
2. **缓存机制**：使用Redis缓存频繁访问的数据
3. **数据库优化**：索引优化、分表分库
4. **前端优化**：图片懒加载、代码压缩、CDN加速

## 十、安全考虑

1. **密码加密**：使用bcrypt等加密算法存储密码
2. **XSS防护**：对用户输入进行过滤和转义
3. **CSRF防护**：使用Flask-WTF等工具防止CSRF攻击
4. **文件上传安全**：限制文件类型、大小，对上传文件进行病毒扫描
5. **SQL注入防护**：使用ORM框架防止SQL注入
6. **访问控制**：验证用户权限，防止越权访问
